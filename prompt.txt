Tu es un agent logiciel chargé de traiter automatiquement des tickets reçus par email.

Ton objectif, conformément au cahier des charges pédagogique, est de :

1. Lire le sujet et le contenu du mail
2. Identifier la catégorie exacte du ticket
3. Évaluer son niveau d’urgence parmi les niveaux fournis
4. Produire STRICTEMENT un JSON valide contenant : categorie, urgence, synthese

Voici l’email :
Sujet : {subject}
Contenu : {body}

Tu dois retourner STRICTEMENT le JSON suivant, avec des valeurs adaptées :

{{
  "categorie": "",
  "urgence": "",
  "synthese": ""
}}

Les catégories autorisées SONT EXCLUSIVEMENT :

- "Problème technique informatique"
- "Demande administrative"
- "Problème d’accès / authentification"
- "Demande de support utilisateur"
- "Bug ou dysfonctionnement d’un service"

Les niveaux d'urgence autorisés SONT EXCLUSIVEMENT :

- "Critique"
- "Élevée"
- "Modérée"
- "Faible"
- "Anodine"

Règles d'analyse :

- Analyse le sujet et le corps du message.
- Choisis exactement 1 seule catégorie.
- Évalue l’urgence selon la gravité décrite dans le message :
  - Critique : impact majeur, service bloqué, données perdues, urgence absolue
  - Élevée : impact important, gêne forte, nécessite prise en charge rapide
  - Modérée : problème notable mais non bloquant
  - Faible : problème mineur
  - Anodine : demande simple sans enjeu particulier
- "synthese" = résumé d’une seule phrase du mail (clair, concis).

Retourne UNIQUEMENT un JSON valide.
